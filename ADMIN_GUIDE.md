# 管理画面の使い方ガイド

湯川寛之オフィシャルサイトの管理画面で、TOPICSや活動報告を簡単に追加・編集できます。

## 📝 基本的な使い方

### 1. TOPICSを追加する

1. 「TOPICS」タブをクリック
2. 「新しいTOPICSを追加」ボタンをクリック
3. 必要な情報を入力:
   - **日付**: TOPICSの日付を選択
   - **カテゴリ**: お知らせ、活動報告、政策提言、イベント、メディア掲載から選択
   - **タイトル**: TOPICSのタイトル（例: 須磨区防災対策について住民説明会を開催）
   - **説明文**: TOPICSの詳細（任意）
   - **画像**: 画像をアップロード（任意）
4. 「保存」ボタンをクリック

### 2. 活動報告を追加する

1. 「活動報告」タブをクリック
2. 「新しい活動報告を追加」ボタンをクリック
3. 必要な情報を入力:
   - **日付**: 活動日を選択
   - **カテゴリ**: 街頭活動、地域懇談会、政策提言、地域イベント、視察、会議から選択
   - **タイトル**: 活動のタイトル（例: 朝の駅前活動 - 通勤通学の皆様と対話）
   - **活動内容**: 活動の詳細を記入
   - **写真**: 複数の写真をアップロード可能
4. 「保存」ボタンをクリック

### 3. 統計情報を更新する

1. 「統計情報」タブをクリック
2. 以下の数値を更新:
   - **地域訪問回数**: 地域を訪問した回数
   - **住民の声聴取数**: 住民から聴取した声の数
   - **地域イベント参加数**: 参加した地域イベントの数
   - **最終更新日**: 統計情報の最終更新日
3. 「保存してサイトに反映」ボタンをクリック

### 4. 変更内容をサイトに反映する

すべての編集が完了したら:

1. **「保存してサイトに反映」ボタン**（画面右上）をクリック
2. `content.json`ファイルが自動的にダウンロードされます
3. このファイルをGitHubにアップロードします:

```bash
# ローカルマシンで実行（サンドボックス不要）
cd Yukawa
cp ~/Downloads/content.json public/data/content.json
git add public/data/content.json
git commit -m "活動報告を更新"
git push origin main
```

4. GitHubにpushすると、約1-2分で本番サイトに自動反映されます！

## 📷 画像のアップロード方法

### 方法1: 直接アップロード（推奨）

1. 管理画面のフォームで「画像をアップロード」エリアをクリック
2. 画像ファイルを選択
3. プレビューが表示されます
4. そのまま保存すれば、画像データがBase64形式でJSONに埋め込まれます

⚠️ **注意**: Base64形式は画像サイズが大きくなるため、1枚あたり500KB以下を推奨

### 方法2: GitHub Issues経由（大きな画像）

大きな画像の場合は、GitHub Issuesにアップロードしてリンクを取得:

1. https://github.com/RyoA12341234/Yukawa/issues を開く
2. 「New issue」をクリック
3. 画像をドラッグ&ドロップ
4. 自動生成されたURL（例: `https://user-images.githubusercontent.com/...`）をコピー
5. 管理画面のフォームに貼り付け

## ✏️ 編集・削除

### TOPICSや活動報告を編集する

1. 各アイテムの右側にある✏️（編集）ボタンをクリック
2. 内容を編集
3. 「保存」ボタンをクリック

### TOPICSや活動報告を削除する

1. 各アイテムの右側にある🗑️（削除）ボタンをクリック
2. 確認ダイアログで「OK」をクリック

## 🎨 カテゴリについて

### TOPICSのカテゴリ
- **お知らせ**: 一般的な告知事項
- **活動報告**: 日々の活動について
- **政策提言**: 政策に関する提言や意見
- **イベント**: イベントの案内や報告
- **メディア掲載**: メディアへの掲載情報

### 活動報告のカテゴリ
- **街頭活動**: 駅前での街頭演説や配布活動
- **地域懇談会**: 地域住民との懇談会
- **政策提言**: 政策提言活動
- **地域イベント**: 地域イベントへの参加
- **視察**: 施設や現場の視察
- **会議**: 各種会議への出席

## 💡 使い方のコツ

### 1. 定期的に更新する
- 週1回は活動報告を追加すると、サイトが活発に見えます
- TOPICSは重要な告知があるたびに追加

### 2. 写真を効果的に使う
- 活動報告には必ず写真を1-3枚追加
- 笑顔の写真、対話している写真が効果的
- 画像サイズは事前に圧縮（推奨: 1枚500KB以下）

### 3. タイトルは具体的に
- ❌ 悪い例: 「今日の活動」
- ✅ 良い例: 「名谷駅前で朝の街頭活動 - 通勤通学の皆様100名以上と対話」

### 4. 統計情報は月1回更新
- 毎月1日に前月の活動をカウント
- 訪問回数、声聴取数、イベント参加数を更新

## 🔧 トラブルシューティング

### Q: 保存ボタンを押しても反映されない
A: 以下を確認してください:
1. ダウンロードした`content.json`をGitHubにpushしましたか？
2. GitHubのリポジトリに正しくファイルがアップロードされていますか？

### Q: 画像が表示されない
A: 以下を確認してください:
1. 画像ファイルは500KB以下ですか？（大きすぎる場合は圧縮）
2. 画像形式はJPGまたはPNGですか？
3. GitHub Issues経由でアップロードしたURLが正しいですか？

### Q: 編集内容が消えた
A: ブラウザのLocalStorageに一時保存されています。
- ブラウザのキャッシュをクリアすると消えます
- 必ず「保存してサイトに反映」ボタンを押してJSONをダウンロードしてください
